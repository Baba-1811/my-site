## はじめに

この記事では、**GitとGitHubを使ったことがない人**を対象に、環境構築から基本操作・実践チュートリアルまでを丁寧に解説します。

この記事を読み終わると、以下ができるようになります。

- Mac に Git をインストールして初期設定できる
- GitHubアカウントを作成してSSH接続できる
- `commit` と `push` の意味と操作を理解できる
- `branch` を使った開発の流れを理解できる
- VSCode から Git 操作ができる
- 実際のプロジェクトで Git を使い始められる

## GitとGitHubを使うメリット

### Gitを使わないとどうなるか

Gitを使わずに開発していると、こんな経験をしたことはありませんか？

- `最終版.py` `最終版2.py` `最終版_本当に最終.py` というファイルが増え続ける
- 昨日まで動いていたコードがなぜか動かなくなり、どこを変えたかわからない
- 友人とコードを共有するとき、毎回ファイルをLINEやメールで送り合う
- 間違えてファイルを上書き保存して、元に戻せなくなる

Gitを使うとこれらが全部解決します。

### Gitでできること

Gitは**バージョン管理システム**です。コードの変更履歴をすべて記録してくれます。

- いつ・誰が・どこを変えたかを記録できる
- 過去の状態にいつでも戻れる
- 複数の機能を並行して開発できる（branch機能）

### GitHubでできること

GitHubは**Gitの履歴をオンラインで管理するサービス**です。

- コードをクラウドにバックアップできる
- チームで同じコードを共同編集できる
- 自分のコードをポートフォリオとして公開できる
- 世界中のオープンソースコードを参照・利用できる

### 研究・開発での具体的な活用例

| 場面 | Gitなし | Gitあり |
|------|---------|---------|
| コードが壊れた | 手動で戻す（または諦める） | 1コマンドで戻せる |
| 新機能を試したい | 元のコードを別名保存 | branchを切って安全に試せる |
| 共同作業 | ファイルを送り合う | GitHubで自動マージ |
| バックアップ | 自分でコピー | pushするだけで自動保存 |

## GitとGitHubとは？

### Gitとは

Gitは**自分のパソコンの中で動くバージョン管理ツール**です。インターネット不要で使えます。コードの変更履歴を「セーブポイント」として記録していくイメージです。

### GitHubとは

GitHubは**Gitの履歴をオンラインで保存・共有できるWebサービス**です。GitHubがなくてもGitは使えますが、GitHubと組み合わせることでバックアップや共同作業が可能になります。

### ローカルとリモートの違い

Gitを使う上で最も重要な概念です。

- **ローカル**：自分のパソコンの中にある作業環境
- **リモート**：GitHubのサーバー上にある保存場所
```
自分のパソコン（ローカル）          GitHub（リモート）
─────────────────────          ──────────────────
コードを書く                        
     ↓ commit                       
履歴を記録する                      
     ↓ push                         
                          →→→  GitHubに送られて保存される
```

`commit` はローカルに履歴を保存すること、`push` はその履歴をGitHubに送ることです。この2つがGitの基本操作の核心です。

## 環境構築

### 1. Homebrewのインストール

HomebrewはMacのパッケージ管理ツールです。Gitを含む多くの開発ツールをコマンド1つでインストールできます。

まずターミナルを開いてください。`Command + Space` で Spotlight を開き、「ターミナル」と入力して Enter を押します。

以下のコマンドをターミナルに貼り付けて実行してください。
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

途中でパスワードを求められたら、Macのログインパスワードを入力してください。入力中は画面に何も表示されませんが、正常です。

インストールが完了したら以下で確認します。
```bash
brew --version
```

`Homebrew 4.x.x` のように表示されれば成功です。

> **すでにHomebrewが入っている場合**
> `brew --version` を実行してバージョンが表示されれば、インストールは不要です。

---

### 2. Gitのインストール

Homebrewを使ってGitをインストールします。
```bash
brew install git
```

インストール後、以下でバージョンを確認します。
```bash
git --version
```

`git version 2.x.x` のように表示されれば成功です。

---

### 3. Gitの初期設定

Gitに自分の名前とメールアドレスを登録します。これはcommitの記録に使われます。GitHubのアカウント情報と合わせておくと管理しやすいです。
```bash
git config --global user.name "あなたの名前"
git config --global user.email "あなたのメールアドレス"
```

設定を確認します。
```bash
git config --list
```

`user.name` と `user.email` が表示されれば設定完了です。

---

### 4. GitHubアカウントの作成

https://github.com にアクセスして「Sign up」からアカウントを作成してください。

- **Username**：英数字で好きな名前（後から変更可能）
- **Email**：先ほど `git config` で設定したメールアドレスと同じものを使うと管理しやすい
- **Password**：安全なパスワードを設定

アカウント作成後、登録したメールアドレスに確認メールが届きます。メール内のリンクをクリックして認証を完了してください。

---

### 5. SSH鍵の設定

SSH鍵を設定すると、GitHubにpushするたびにパスワードを入力しなくてよくなります。

**SSH鍵を生成する**
```bash
ssh-keygen -t ed25519 -C "あなたのメールアドレス"
```

実行すると以下のように聞かれます。
```
Enter file in which to save the key (/Users/yourname/.ssh/id_ed25519):
```

そのまま Enter を押します（デフォルトの場所に保存）。
```
Enter passphrase (empty for no passphrase):
```

パスフレーズを設定する場合は入力します。設定しない場合はそのまま Enter を2回押します。

**公開鍵をコピーする**
```bash
pbcopy < ~/.ssh/id_ed25519.pub
```

これでクリップボードに公開鍵がコピーされました。

**GitHubに公開鍵を登録する**

1. GitHubにログインして右上のアイコンをクリック
2. `Settings` を選択
3. 左メニューの `SSH and GPG keys` をクリック
4. `New SSH key` をクリック
5. `Title` に「My MacBook」など識別しやすい名前を入力
6. `Key` に先ほどコピーした公開鍵を貼り付け
7. `Add SSH key` をクリック

**接続を確認する**
```bash
ssh -T git@github.com
```

以下のように表示されれば成功です。
```
Hi ユーザー名! You've successfully authenticated, but GitHub does not provide shell access.
```

> **よくあるエラー**
> `Permission denied (publickey)` と表示された場合は、SSH鍵の登録が正しくできていない可能性があります。手順4〜6をもう一度確認してください。
